# BGP 知识库目录介绍

## 项目概览

当前远端交互式部署位于 `root@10.99.8.28:/home/wbt/DB`，其中知识库后端在 `/home/wbt/DB/bgp_knowledge_base`，对话前端在 `/home/wbt/DB/chat_frontend`。前端通过 `http://10.99.8.28/` 或 `http://10.99.8.28:39280/` 访问，FastAPI 后端运行在 `39281`，embedding/reranker 分别运行在 `8011/8012`。服务当前使用 `screen` 常驻运行，操作命令统一见 `docs/operations/remote_server_operations_v1.md`。

```
bgp_knowledge_base/
├── data/             数据资产、派生产物、发布包、报告和人工复核输入
├── deploy/           私有模型服务与离线运行时部署脚本
├── metadata/         配置、分类体系和 JSON Schema
├── src/bgpkb/        流水线模块和只读服务代码
├── tests/            自动化测试
├── docs/             规划、治理、阶段和规则文档
├── README.md         项目总览
└── 目录介绍.md       本文件
```

数据流向：

```
data/sources/raw
  → data/corpus/parsed
  → data/corpus/cleaned + data/corpus/cleaned_blocks_v2
  → data/corpus/chunks + data/corpus/chunks_v2 + data/derived/datasets/section_catalog.jsonl
  → data/knowledge/entities + data/knowledge/relationships
  → data/derived/datasets + data/published + data/reports + data/generated/reports
```

## 上级目录分区

| 分区 | 内容 | 是否人工编辑 |
| --- | --- | --- |
| `data/sources/` | 原始资料和来源登记表。 | `raw/` 不改；`inventory/` 可人工维护。 |
| `data/corpus/` | 解析文本、v1 清洗文本、v2 Canonical Block、v1/v2 chunk。 | 通常由流水线生成，清洗语料可人工校订。 |
| `data/knowledge/` | 实体和关系核心知识资产。 | 可人工维护，应用决策脚本会显式改写实体。 |
| `data/derived/` | 工作簿、队列、评测、清单、section catalog 等派生数据。 | 通常由流水线生成。 |
| `data/review_inputs/` | 人工复核决策输入和 session 模板。 | 人工填写，流水线不得覆盖主决策。 |
| `data/published/` | SQLite、目录、索引、JSON-LD、RAG 索引、manifest 等发布包。 | 由流水线生成。 |
| `data/reports/` | 门禁、参考和行动入口报告。 | 少量人工入口，由报告策略保护。 |
| `data/generated/reports/` | 流水线明细报告、指南和快照。 | 可再生成输出。 |
| `metadata/` | 配置和 Schema。 | 可人工维护。 |
| `src/bgpkb/` | Python 流水线和服务代码。 | 开发维护。 |
| `deploy/` | Docling、retrieval model service 等部署脚本。 | 开发维护；凭据和本机 `.env` 不入库。 |

## metadata/config/ — 配置与分类体系（5 个 YAML 文件）

定义知识库的元规则，是流水线的配置文件。

| 文件 | 作用 | 何时用 |
|---|---|---|
| `source_types.yaml` | 来源类型枚举（standard / paper / data_doc / case_report / blog / manual_note） | 登记新来源时参考 |
| `topic_taxonomy.yaml` | BGP 主题分类体系（BGP基础 → 路径与属性 → BGP安全 → ...） | 给 chunk 或实体打标签 |
| `entity_types.yaml` | 9 种实体类型定义（BGPConcept / AnomalyType / DataSource / ...） | 创建新实体时参考 |
| `quality_rules.yaml` | 质量检查规则（每个实体类型必须有哪些字段、9 项全局检查） | 了解质量门禁 |
| `storage_formats.yaml` | 各数据层存储格式约定 | 了解数据规范 |

**如何检查**：打开任意 YAML 文件阅读即可，全是纯文本配置。

## data/sources/raw/ — 原始资料层（53 个文件，不可修改）

所有外部资料的原始形式，按来源类型分目录。这一层**只读不写**，保证追溯。

| 子目录 | 数量 | 内容 |
|---|---|---|

| `data_docs/` | 14 个 | 数据源文档（RouteViews API、RIPE RIS、CAIDA、RIPEstat、PeeringDB、MANRS 等） |
| `papers/` | 13 个 | 学术论文 PDF（BEAR、BEAM、BGPShield、ARTEMIS、BGP2Vec、ASwatch 等） |
| `cases/` | 13 个 | 案例报告 HTML/PDF（YouTube 劫持、Facebook 宕机、Cloudflare 事件等） |
| `tools_docs/` | 2 个 | 工具文档（BGPStream） |
| `notes/` | 1 个 | 项目上下文笔记 |

**如何检查**：直接打开文件确认内容完整性和格式正常。比如：

```bash
head -50 data/sources/raw/standards/rfc4271.txt
open data/sources/raw/papers/bear_2025.pdf
```

## data/corpus/parsed/ — 解析文本层（53 个 JSON 文件）

从 raw 确定性解析出的结构化文本，保留了章节层级、页码、图表标题等结构信息。

**关键字段**：`doc_id`, `title`, `sections[].heading`, `sections[].content`, `sections[].page_start/end`, `source_path`

**如何检查**：

```bash
python3 -m json.tool data/corpus/parsed/papers/bear_2025.json | head -80
python3 -m json.tool data/corpus/parsed/standards/rfc4271.json | head -50
```

## data/corpus/cleaned/ 与 cleaned_blocks_v2/ — 清洗文本层

`data/corpus/cleaned/` 保留 54 个面向人工阅读的 Markdown 清洗文本；`data/corpus/markdown_v2/` 保留 v2 Markdown 快照；`data/corpus/cleaned_blocks_v2/` 保留 Docling Canonical Block v2、转换审计、验证结果和复核队列，是阶段 B 层级 chunk 的权威输入。

**如何检查**：

```bash
head -100 data/corpus/cleaned/papers/bear_2025.md
head -100 data/corpus/cleaned/standards/rfc4271.md
```

## data/corpus/chunks/ 与 chunks_v2/ — 知识片段层

`data/corpus/chunks/` 是早期 v1 知识片段，合计 2,037 条，仍用于历史评测和兼容入口。阶段 B 发布路径使用 `data/corpus/chunks_v2/*.jsonl`，合计 58,560 条层级 chunk，全部具备 `parent_section_id`、前后邻接和层级状态。

| 文件 | 条数 | 内容 |
|---|---|---|
| `bgp_chunks.jsonl` | 890 | 通用 BGP 概念、数据源说明、工具文档的 chunk |
| `standard_chunks.jsonl` | 554 | RFC 标准文档按 section 切分的 chunk |
| `paper_chunks.jsonl` | 504 | 论文方法按 section 切分的 chunk |
| `case_chunks.jsonl` | 89 | 案例报告按事件要素切分的 chunk |
| `seeds/context_chunks.jsonl` | — | 从 context.md 切出的 seed chunk（提供项目范围和知识边界） |

阶段 B 关键派生物：

| 文件 | 条数 | 内容 |
| --- | ---: | --- |
| `data/corpus/chunks_v2/*.jsonl` | 58,560 | 从 Canonical Block v2 派生的层级 chunk。 |
| `data/derived/datasets/section_catalog.jsonl` | 447 | section catalog，供父 section、邻接窗口和 context pack 回扩使用。 |

**每条 chunk 的结构**：

```json
{
  "chunk_id": "bear_2025_sec3_2_001",
  "doc_id": "bear_2025",
  "chunk_type": "paper_method",
  "topics": ["BGPStream", "RIB", "AS_PATH"],
  "content": "...",
  "source_ref": "BEAR.pdf:p4:III-B",
  ...
}
```

**如何检查**：

```bash
# 统计各类 chunk 数量
wc -l data/corpus/chunks/*.jsonl

# 抽查一条
head -1 data/corpus/chunks/bgp_chunks.jsonl | python3 -m json.tool

# 全文搜索（通过 SQLite）
python3 -m bgpkb.pipeline.query_knowledge_base search-chunks '"route leak"' --limit 5
```

## data/knowledge/entities/ — 结构化实体层（112 条 JSONL，核心资产）

知识库的核心产出。所有 BGP 领域知识被组织成 9 种实体类型，每种实体都有固定的 JSON Schema。

| 文件 | 条数 | 审批状态 | 内容 |
|---|---|---|---|
| `bgp_concepts.jsonl` | 31 | ✅ 31 approved | BGP 基础概念（AS, Prefix, AS_PATH, RPKI, ...） |
| `data_fields.jsonl` | 32 | ✅ 32 approved | BGP 数据字段语义（prefix, as_path, roa_status, ...） |
| `routing_mechanisms.jsonl` | 12 | ✅ 12 approved | 路由机制（Path-vector, Valley-free, ROV, ASPA, BGPsec, ...） |
| `data_sources.jsonl` | 9 | ✅ 9 approved | 外部数据源说明（RouteViews, RIPE RIS, BGPStream, CAIDA, ...） |
| `anomaly_types.jsonl` | 8 | ✅ 8 approved | 异常类型定义（Hijack, Route Leak, MOAS, Outage, ...） |
| `evidence_templates.jsonl` | 8 | ✅ 8 approved | 每类异常需要的证据模板 |
| `cases.jsonl` | 5 | ✅ 1 approved + 4 needs_source | 历史 BGP 事件案例 |
| `false_positive_patterns.jsonl` | 4 | ✅ 4 approved | 误报与边界模式 |

## data/knowledge/relationships/ — 关系组织层（106 条 JSONL）

描述实体之间的语义关系（如 `requires_evidence`、`supports`、`belongs_to`、`provides` 等）。

**关系类型举例**：

- `AnomalyType → requires_evidence → EvidenceTemplate`
- `BGPConcept → supports → AnomalyType`
- `DataSource → provides → DataField`
- `FalsePositivePattern → guards_against_overclaiming → AnomalyType`

**如何检查**：

```bash
# 查某个实体的所有关系邻居
python3 -m bgpkb.pipeline.query_knowledge_base neighbors concept_as_path
python3 -m bgpkb.pipeline.query_knowledge_base neighbors anomaly_route_leak

# 直接看文件
head -5 data/knowledge/relationships/relationships.jsonl | python3 -m json.tool
```

## src/bgpkb/pipeline/ — 流水线脚本（76 个 Python 文件）

所有数据处理、质量检查和发布脚本。统一入口：

```bash
python3 -m bgpkb.pipeline.run_pipeline        # 一键运行全部确定性流水线
python3 -m bgpkb.pipeline.query_knowledge_base # 查询入口
```

**核心脚本分类**：

| 类别 | 脚本 | 作用 |
|---|---|---|
| 解析 | `parse_documents.py` | 从 raw 解析为 parsed JSON |
| 清洗 | `clean_text.py` | 从 parsed 清洗为 cleaned Markdown |
| 切分 | `build_chunks.py` | 从 cleaned 切分为 chunks JSONL |
| 实体 | `extract_entities.py` | 从 chunks 抽取实体（当前为人工整理入口） |
| 关系 | `build_relationships.py` | 构造实体关系图 |
| 质量 | `quality_check.py` | 全面质量校验 |
| 案例 | `extract_case_observations.py` | 从案例 chunk 机械抽取观察值 |
| 人工复核 | `build_human_review_*.py`（20 个） | 生成工作簿/模板/校验/审计/进度/摘录/会话队列/字段清单/来源矩阵/任务板/交接清单/指南 |
| 应用 | `apply_human_review_decisions.py` | 将人工复核决策写入实体文件 |
| 发布 | `build_published_knowledge_base.py` | 生成 data/published/ 发布包 |
| 发布 | `build_sqlite_knowledge_base.py` | 生成 SQLite 数据库 |
| 发布 | `build_published_integrity_report.py` | 校验 SQLite vs manifest 一致性 |
| 查询 | `query_knowledge_base.py` | CLI 查询入口（stats / entity / neighbors / evidence / search-chunks / glossary 等） |
| RAG | `build_rag_indexes.py`、`build_bge_m3_index.py`、`run_hybrid_retrieval_eval.py`、`evaluate_chunking.py` | 构建检索索引、BGE-M3 向量索引、混合检索评测和阶段 B chunking 评测 |
