# ADR 0006：RAG Recall@8 发布阈值采用 0.80

- 状态：已批准
- 日期：2026-07-15
- 决策人：项目负责人（本地开发会话明确批准）
- 适用策略：`rag_quality_gates_v1.1.0`

## 背景

首版门禁要求 Recall@8 不低于 0.90，且相对冻结基线最多回退 0.02。候选真实评测得到 0.8125。继续围绕当前黄金集提升该指标时，已经出现把宽泛问题直接扩展到黄金集指定来源、复用黄金负例固定词组，以及用扩展结果强制来源入选等明显过拟合行为。

这些调整会提高单一黄金集分数，却不能证明对未见问题的泛化能力，反而可能压低真实用户问题的精排质量与拒答可靠性。

## 决策

将 Recall@8 硬门禁下限设为 0.80，相对冻结基线允许的最大回退设为 0.10。两条规则仍同时生效；当前冻结基线保持 0.882353，不重写历史事实。

同时规定：

1. 不允许把宽泛语义直接扩展成黄金集指定数据源、事件或论文名称。
2. 来源锚点只识别用户原始问题中明确出现的来源名，不读取查询扩展结果。
3. 拒答由证据能否直接支持问题中的事实、关系或操作决定，不使用与黄金负例一致的固定词表。
4. 继续保留 MRR、引用精度、claim 引用覆盖率、困难负例、提示注入与真实性能门禁，Recall@8 达标不能替代其他门禁。

## 影响

该决策接受一定召回上限损失，以换取更低的黄金集耦合和更可解释的线上行为。0.80 是发布最低标准，不是优化目标；后续提升应通过新来源、切块质量、通用检索模型或盲测集验证完成。

## 验证与监控

- 每次候选发布使用冻结的 100 题以上检索集重新计算 Recall@8 与 MRR。
- 定期增加未参与调参的盲测问题，并按来源类型、语言和问题类型分层报告。
- 记录显式来源查询与宽泛查询的分项召回，确认来源锚点没有从扩展词注入。
- 任一答案门禁、真实性能门禁或完整性门禁失败时仍阻断发布。

## 回滚

如果盲测或线上审计显示 0.80 导致不可接受的证据缺失，应发布新的版本化策略，将阈值恢复到更高水平，并通过新的 ADR 记录数据依据。不得静默修改本 ADR 或覆盖既有评测报告。
