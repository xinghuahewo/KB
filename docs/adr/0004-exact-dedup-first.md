# ADR-0004：首版只启用同源精确去重硬门禁

## 状态

已接受，2026-07-14。

## 背景

生产语料的规范化预览重复冗余率为 75.5447%，但跨来源相似论述可能代表独立证据，不能因文本接近而删除。迁移 fixture 对 exact hash、token shingles、64 次确定性 MinHash 和 64-bit SimHash 做了同源/跨来源黄金对比较。

## 比较结果

| 策略 | 误折叠率 | 漏折叠率 | 说明 |
| --- | ---: | ---: | --- |
| exact hash | 0% | 50% | 不折叠数字不同的模板近重复 |
| token 3-shingles | 0% | 0% | 数字归一化后可识别 fixture 模板 |
| MinHash | 0% | 0% | 当前小样本与 shingles 一致 |
| SimHash | 0% | 0% | 跨来源相似度高，但来源边界阻止折叠 |

当前黄金集规模不足以证明近重复算法在 RFC、自然语言和 OpenAPI 全量语料上的误折叠安全性。尤其 SimHash 对两个独立来源的 ROV 表述相似度达到 0.890625；若遗漏来源边界，将有误删独立证据的风险。

## 决策

- 首版生产硬门禁只自动折叠同一 source snapshot、相同语义上下文下的规范化 exact hash。
- token shingles、MinHash 和 SimHash 只输出诊断，不自动删除或合并。
- 跨来源内容永不在语料层自动折叠，只允许检索结果层抑制重复展示。
- 后续启用近重复折叠必须扩充黄金集、给出误折叠样例、阈值和人工批准，并新增 ADR。

## 后果

首版仍会保留部分同源模板近重复，但不会以追求更低重复率为代价破坏证据独立性。实际策略由 `dedup_strategy_evaluation` 的确定性结果和版本化配置共同追溯。
