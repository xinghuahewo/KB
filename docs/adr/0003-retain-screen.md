# ADR-0003：现阶段保留 screen

- 状态：临时接受
- 日期：2026-07-13

## 背景

现有前端和 FastAPI 通过固定名称的 screen 会话稳定运行，端口和巡检流程已经验证。立即迁移 systemd 会扩大本次目录与数据重构风险。

## 决策

本次重构保留 screen 会话名和端口契约，先建立可靠的 release、deploy、rollback 和健康检查入口。稳定一个发布周期后再评估运行管理器。

## 后果

迁移风险较低，但服务自愈、权限隔离和声明式配置仍有限。运维文档必须保留明确的启动、巡检和回滚步骤。

## 移除条件

统一部署入口稳定、至少一次联合回滚演练通过，并明确 systemd 或容器运行方案后，创建新 ADR 替代本决策。
