# ADR-0001：采用模块化单体

- 状态：已接受
- 日期：2026-07-13

## 背景

知识库包含离线数据处理、在线检索、API 和前端，但团队规模与部署环境尚不需要微服务拆分。旧代码按阶段脚本增长，边界不够清晰。

## 决策

保留单仓库和单后端部署单元，在 Python 包内部建立 domain、infrastructure、ingestion、indexing、publishing、retrieval、api、workflows 边界，并用依赖测试约束方向。

## 后果

部署和本地开发保持简单；模块可独立测试。代价是必须持续维护边界，避免 workflows 和 API 重新吸收领域逻辑。

## 未采用方案

- 微服务：当前会增加部署、网络和一致性成本。
- 保持阶段脚本平铺：无法形成稳定公共接口和依赖方向。
