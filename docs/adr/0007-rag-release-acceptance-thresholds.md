# ADR 0007：首版真实候选的 RAG 验收阈值

- 状态：已批准
- 日期：2026-07-15
- 决策人：项目负责人（本地开发会话明确批准“差不多得了，通过吧”）
- 适用策略：`rag_quality_gates_v1.2.0`
- 前置决策：[ADR 0006：RAG Recall@8 发布阈值采用 0.80](0006-rag-recall-threshold.md)

## 背景

候选在目标服务器隔离 canary 上完成 104 道检索题和 24 道结构化回答题的真实评测。报告绑定候选 manifest、代码提交、BGE-M3、真实 reranker 和 DeepSeek V4 Pro，技术执行无请求失败、无模型降级，数据、制品和 API 契约门禁通过。

真实指标为 Recall@8 0.829545、MRR 0.621550、claim 引用覆盖率 0.40625、引用精度 0.625、困难负例拒答率 1.0、提示注入防护率 0.75。继续针对当前黄金集提升分数已经出现概念到指定来源的定向扩展风险，因此项目负责人决定接受当前真实能力作为首版验收线，不再围绕单一黄金集调参。

## 决策

发布策略升级为 `rag_quality_gates_v1.2.0`，硬门禁下限为：

| 指标 | 下限 |
| --- | ---: |
| Recall@8 | 0.80 |
| MRR | 0.60 |
| claim 引用覆盖率 | 0.40 |
| 引用精度 | 0.60 |
| 困难负例拒答率 | 1.00 |
| 提示注入防护率 | 0.75 |

数据完整性、快索引、无降级和检索性能门禁保持不变。所有指标仍以真实候选报告计算，不改写逐题结果，不把失败样本伪装成通过样本。

## 约束

1. 不恢复宽泛概念到黄金来源、事件或论文名称的定向扩展。
2. 不恢复天气、股票、手机号等黄金负例固定词表。
3. 来源锚点只处理用户原始问题中明确出现的来源名。
4. 后续提升必须使用新增盲测集、通用模型或数据质量改进证明，不能只优化当前黄金集。
5. 本阈值是首版最低验收线，不代表产品质量目标。

## 影响与后续

该决策允许当前候选进入隔离 canary 和后续人工发布评审，同时显式接受答案证据覆盖仍有改进空间。后续版本应分层报告中英文、问题类型和来源类型，并在至少一套未参与调参的盲测集上提高阈值。

## 回滚

若 canary 或生产审计出现错误引用、越界回答或注入风险，立即执行代码/制品成对回滚；候选报告和本 ADR 保留。恢复发布时必须发布新的策略版本和新 ADR，不得静默降低本版本阈值。
